# Accessibility Review — Volume 1

Status: Open

Review the exact HTML, EPUB, and DOCX files proposed for release. Passing a
build does not establish accessibility conformance.

## HTML

- [ ] Descriptive document title
- [ ] Logical heading hierarchy
- [ ] Keyboard navigation and visible focus
- [ ] Meaningful link text
- [ ] Sufficient text and interface contrast
- [ ] Tables with understandable headers
- [ ] Text equivalents for meaningful diagrams
- [ ] Useful reading order at narrow and wide widths

## EPUB

- [ ] Navigation document and table of contents
- [ ] Language and title metadata
- [ ] Accessibility metadata required by the chosen conformance target
- [ ] Reflow at enlarged text sizes
- [ ] Heading, list, link, and table semantics
- [ ] Diagram alternatives available without images

## DOCX

- [ ] Built-in heading styles
- [ ] Logical reading order
- [ ] Understandable tables
- [ ] Meaningful links
- [ ] Diagram alternatives
- [ ] Document language and title properties

## Assistive Review

- [ ] Test HTML with keyboard-only navigation.
- [ ] Test at least one output with a screen reader or equivalent tool.
- [ ] Test EPUB in at least two reading applications.
- [ ] Record defects, accepted exceptions, owners, and correction dates.

## Reference
