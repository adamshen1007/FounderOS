# RFC-000 — FounderOS RFC Process

## Status

Accepted

## Purpose

The RFC process is used to propose significant changes before implementation.

## When to Use an RFC

Use an RFC for:
- New major modules
- New MCP/tool integrations
- Changes to source-of-truth rules
- Publishing pipeline changes
- AI agent responsibility changes
- Governance or quality gate changes

## RFC Lifecycle

1. Draft
2. Review
3. Accepted / Rejected / Deferred
4. Implementation
5. Retrospective

## Required Sections

- Summary
- Motivation
- Proposal
- Alternatives considered
- Risks
- Acceptance criteria
- Implementation plan
