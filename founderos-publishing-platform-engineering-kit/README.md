# FounderOS Publishing Platform

FounderOS Publishing Platform is the documentation-first foundation for building an AI-native founder operating system.

It uses Git + Markdown as the source of truth, Obsidian for knowledge management, Notion for editorial workflow, AI agents for drafting and automation, and CI/CD for validation and publishing.

## Repository Principles

- Documentation before implementation
- Research before commitment
- Milestone-scoped execution
- Markdown as canonical source
- Notion, PDF, EPUB, DOCX, and websites as generated outputs
- Human approval for major strategic and architectural decisions

## Initial Structure

- `/governance` — constitution, RFCs, ADRs, policies
- `/docs` — human-readable operating guides
- `/specs` — implementation contracts
- `/templates` — reusable book, chapter, RFC, ADR, and worksheet templates
- `/prompts` — reusable AI and Codex prompts
- `/books` — source content for Founder Library volumes
- `/research` — raw and processed research
- `/publishing` — output pipeline configuration
- `/ci` — quality gates and build checks
