# FounderOS AI Agent Instructions

## Universal Rules

- Markdown is the source of truth.
- Do not invent private or unsupported facts.
- Clearly distinguish sourced facts, synthesis, and assumptions.
- Prefer small, milestone-scoped changes.
- Update relevant specs/docs when behavior changes.
- Use RFCs for major changes.
- Use ADRs for accepted architectural decisions.

## Required Chapter Elements

Every book chapter should include:

- Learning objectives
- Core principle
- Deep-dive explanation
- AI founder interpretation
- Callout boxes
- Diagrams where helpful
- Checklist
- Worksheet
- Key takeaways
- Sources section when external claims are used
