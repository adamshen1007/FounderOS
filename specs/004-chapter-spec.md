# Chapter Specification

Each book chapter must include:

- A level-one title
- Learning objectives
- A core principle
- A deep-dive explanation
- An AI founder interpretation
- A callout
- Diagrams where helpful
- A checklist
- A worksheet
- Key takeaways
- A Sources section when external claims are used

Chapter filenames must begin with a zero-padded sequence number so build order
is deterministic, for example `01-why-startups-win.md`.

Sources must use public Markdown links. Placeholders such as `TODO`, `TBD`,
`citation needed`, and `add sources` fail the M1 citation gate.
